# Java Cache Integration Tests

## Setup (run once)

```bash
# 1. Install Dart dependencies
dart pub get

# 2. Generate the Java client from the schema
make generate

# 3. Build the Spring Boot server jar (shared with Dart cache tests)
make build-server
```

## Running the tests

```bash
# Option A: start the server manually in a separate terminal, then run tests
make run-server          # terminal 1
make test                # terminal 2

# Option B: the test suite starts and stops the server automatically
make test
```

## Rules

### Generated Code — NEVER TOUCH
Never read, edit, create, or delete any file under `client-app/src/main/java/com/example/generated/`.
These files are auto-generated by GraphLink and any manual change will be lost on the next generation.

### Schema consistency
`schema/schema.gql` (client, with graphlink directives) and
`../cache_tests/spring-server/src/main/resources/graphql/schema.graphqls` (server, plain GraphQL)
must declare the same types, queries, and mutations.
If you change one, update the other.

### Shared Spring server
This test suite reuses the Spring Boot server from `../cache_tests/spring-server/`.
Run `make build-server` once (or after any schema change) before running tests.
